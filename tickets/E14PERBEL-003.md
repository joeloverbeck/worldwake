# E14PERBEL-003: Register AgentBeliefStore and PerceptionProfile in ComponentTables

**Status**: PENDING
**Priority**: HIGH
**Effort**: Small
**Engine Changes**: Yes — component registration in ECS layer
**Deps**: E14PERBEL-002 (types must exist before registration)

## Problem

`AgentBeliefStore` and `PerceptionProfile` need to be registered in the component schema so they can be stored on entities via `World` and `WorldTxn` APIs. Without registration, no system can attach, query, or mutate these components on agent entities.

## Assumption Reassessment (2026-03-14)

1. Component registration uses the `with_component_schema_entries!` macro in `component_schema.rs` — confirmed. 28 components currently registered.
2. Each registration generates: table field, insert/get/get_mut/remove/has/iter methods on `ComponentTables`, and corresponding `WorldTxn` methods — confirmed by macro pattern.
3. Both components are `EntityKind::Agent`-only — spec says "registered for `EntityKind::Agent`".
4. The `ComponentTables` struct in `component_tables.rs` is auto-generated from the macro — confirmed.
5. `WorldTxn` methods are generated by the same macro infrastructure — confirmed.

## Architecture Check

1. Follows the exact same pattern as all 28 existing components — no special handling needed.
2. EntityKind guard restricts these components to agents only — prevents misuse on places/items.

## What to Change

### 1. Add entries to `component_schema.rs`

Add two new entries to the `with_component_schema_entries!` macro following the exact pattern of existing agent-only components (e.g., `CombatProfile`, `HomeostaticNeeds`):

- `AgentBeliefStore` — field name `agent_belief_store`, registered for `EntityKind::Agent`
- `PerceptionProfile` — field name `perception_profile`, registered for `EntityKind::Agent`

Each entry needs:
- Field name, type, table methods (insert, get, get_mut, remove, has, iter)
- Txn methods (insert, get, get_mut, remove, has, entities, query, count)
- Display name string
- EntityKind predicate: `|kind| kind == EntityKind::Agent`
- ComponentKind/ComponentValue variant names

### 2. Add ComponentKind/ComponentValue variants

If `ComponentKind` and `ComponentValue` enums are separate from the macro (check), add `AgentBeliefStore` and `PerceptionProfile` variants.

### 3. Update `canonical.rs` if needed

If `canonical_bytes()` or `hash_world()` explicitly enumerates component types for hashing, add the new components. Check whether the macro handles this automatically.
