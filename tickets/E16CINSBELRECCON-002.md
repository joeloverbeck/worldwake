**Status**: PENDING

## Problem

Records must be first-class world entities. This requires adding `EntityKind::Record` to the entity classification, registering `RecordData` in the component schema/tables, extending the delta system with `ComponentKind::RecordData` / `ComponentValue::RecordData`, and adding canonical hashing support.

## What to Change

### 1. `EntityKind::Record` in `entity.rs`

Add `Record` variant after `Rumor`. Update `ALL_ENTITY_KINDS` test array from 10 to 11 entries. Ensure ordering test still passes (variants must be in sorted `Ord` order).

### 2. Component table storage in `component_tables.rs`

Add `record_data: BTreeMap<EntityId, RecordData>` via the `with_component_table_entries!` macro.

### 3. Component schema in `component_schema.rs`

Register `RecordData` with kind predicate `kind == EntityKind::Record`.

### 4. Delta system in `delta.rs`

Add `RecordData` to the macro that generates `ComponentKind::RecordData` and `ComponentValue::RecordData(RecordData)`. Add a sample `RecordData` to the `component_samples()` test helper.

### 5. Canonical hashing in `canonical.rs`

Add `RecordData` to the canonical hashing traversal so `hash_world()` covers record entities.

### 6. World creation helper

Add `World::create_record()` convenience method (allocates entity with `EntityKind::Record`, attaches `RecordData` component). Mirror pattern of existing `create_*` methods.
