# E16DPOLPLAN-001: Add `courage()` to `RuntimeBeliefView` trait + implementation

**Status**: ✅ COMPLETED

**Deps**: None

## Problem

The GOAP planner needs to read target courage during Threaten evaluation, but `RuntimeBeliefView` has no `courage()` method.

## Assumption Reassessment (2026-03-18)

1. `RuntimeBeliefView` trait is in `crates/worldwake-sim/src/belief_view.rs` — confirmed
2. ~~`OmniscientBeliefView` is in `crates/worldwake-sim/src/omniscient_belief_view.rs`~~ — **INCORRECT**: no such struct or file exists. Only `PerAgentBeliefView` implements `RuntimeBeliefView`.
3. `PerAgentBeliefView` is in `crates/worldwake-sim/src/per_agent_belief_view.rs` — confirmed
4. `UtilityProfile.courage` field exists in `crates/worldwake-core/src/utility_profile.rs` — confirmed
5. ~~`combat_profile()` method exists as default method pattern to follow~~ — **CORRECTED**: `combat_profile()` is a required method, not a default. The default-method pattern to follow is `tell_profile()`, `office_data()`, `loyalty_to()`, etc.

## Architecture Check

1. Default method with `None` return follows existing trait pattern (see `tell_profile()`, `office_data()`, `loyalty_to()`)
2. `PerAgentBeliefView` impl gates on `agent == self.agent` (self-authoritative read pattern, same as `combat_profile` impl at line 504)

## What to Change

### 1. `RuntimeBeliefView` trait — add default method

Add after `combat_profile()` (~line 188):
```rust
fn courage(&self, agent: EntityId) -> Option<Permille> {
    let _ = agent;
    None
}
```

### 2. `PerAgentBeliefView` — implement

```rust
fn courage(&self, agent: EntityId) -> Option<Permille> {
    (agent == self.agent)
        .then(|| self.world.get_component_utility_profile(agent).map(|p| p.courage))
        .flatten()
}
```

### 3. `impl_goal_belief_view` macro — add delegation

Add delegation entry for `courage()` so `GoalBeliefView` impls generated by the macro forward to `RuntimeBeliefView::courage`.
